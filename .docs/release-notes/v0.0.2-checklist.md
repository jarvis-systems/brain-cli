## Rollback Plan

If critical issues are found after merge:

### 1. Revert all release commits (reverse order)

```bash
cd cli
git revert 5309d19 --no-edit   # docs fix
git revert 7c15655 --no-edit   # post-merge checklist
git revert 6fe0a1f --no-edit   # main release commit
# Creates 3 revert commits. Preserves history.
```

### 2. Delete tag (local + remote)

```bash
git tag -d v0.0.2
git push origin :refs/tags/v0.0.2
```

### 3. Push reverts to remote

```bash
git push origin master
```

### 4. Restore previous version

```bash
# composer.json already reverted by step 1
git tag -l  # Confirm v0.0.1 is still present
```

### 5. Verify rollback

```bash
composer test     # Must still pass (reverted to pre-refactor state)
composer analyse  # Must still pass
bin/brain --version  # Should show v0.0.1
```

### Parent repo CI (no rollback needed)

The brain-lint.yml change (commit 3635d3f) is backward-compatible.
`composer test` in cli/ passes on both pre- and post-refactor code.
No revert needed in parent repo.
